# ADR-0006：按证据能力而非单一二元门禁资格化转录数据

- 状态：Accepted
- 日期：2026-07-15
- 关系：细化并部分取代 [ADR-0005](ADR-0005-evidence-integrated-boundary-track.md) 中“单一来源整体通过后才能继续”的解释；ADR-0005 的主坐标、停止线和边界轨道目标继续有效。
- 后续修订：[ADR-0007](ADR-0007-srr31989028-controlled-probe-outcome.md) 将 acquisition 实现改为完整 normalized SRA，并记录受控探针失败结论。[ADR-0008](ADR-0008-target-strain-native-sequencing-redirect.md) 因 Strain-T 原生测序获得授权而暂停本 ADR 定义的下一步公开数据路线（能力分级方法论本身继续有效，仅数据源改变）。

## 背景

Slice 0B 及两个后续单一来源调查表明：

- `PRJNA311606` 与当前 assembly/annotation 生成过程重叠，不能作为独立证据；
- `PRJNA604658` 时间独立，但样本表达 eGFP、只有一个 WT run、链特异性未声明；
- `PRJNA1210090` 的三个选定 run 明确标记为无胁迫 wild-type Strain-B，具备三个独立 BioSample 和生物重复，但 BioProject 同时描述工程化香草醛细胞工厂，未改造身份仍需序列核查；文库未声明 first/second-strand；公共仓储条款没有逐 FASTQ 独立许可证文本。

如果把逐文件许可证、链方向和未改造身份全部设为来源整体的先验硬失败，公共 SRA/ENA 数据会被系统性挡在受控质量核查之前。另一方面，直接放宽为“可用”又会把不具备的方向、菌株纯净性和再分发权包装成已知事实。

## 决策

转录数据改为按其能够支持的科学能力分别资格化，不再用一个布尔值代表所有用途。

### 1. 科学能力分层

- `coordinate_coverage`：是否能在锁定主坐标上提供可重复覆盖；不要求链特异性。
- `splice_support`：是否能通过受控比对提供可重复剪接连接证据；不要求链特异性，但必须报告多重映射和非典型连接。
- `strand_support`：只有文库方向明确并经验证时才可用；否则保持 `unavailable`。
- `boundary_support`：必须由覆盖、剪接、同源/蛋白证据和冲突规则共同决定；单一 RNA-seq 来源不能独立冻结边界。
- `strain_specific_support`：只有菌株身份和工程背景通过时才可声明；否则最多作为 Strain-B 近似结构证据，不能用于 Strain-T 特异结论。

### 2. 公共数据使用治理

- NCBI/ENA 的公开访问与仓储政策可支持本项目本地计算和可重复性核验，但记为 `archive-terms-with-submitter-caveat`，不解释为逐文件专有许可证。
- 原始 FASTQ/SRA 不进入项目发布包，不再分发；正式产物只记录 accession、来源 URL、大小、MD5/SHA-256、获取日期和政策快照。
- 如果来源明确限制计算使用，则停止；如果只是缺少逐文件许可证文本，不再作为科学质量的自动硬失败，但必须保留治理警告。

### 3. 工程背景处理

- BioProject 描述工程化研究而选定 BioSample 明确标记 wild type 时，状态为 `identity_needs_sequence_check`，不是自动通过，也不是自动淘汰。
- 受控探针必须检查非参考/外源序列、异常覆盖和疑似整合影响；无法排除的区域标记为不可用于菌株特异边界。
- 工程背景来源不得用于表达强度、适应性或 Strain-T 特异推断。

### 4. PRJNA1210090 的当前资格

`PRJNA1210090` 仍未通过科学验收，因为尚未下载原始 reads 或执行受控比对；但它通过了时间独立性、生物重复、条件绑定和 archive Strain-B 标记门禁，允许进入最小受控探针。

## 下一切片：单重复受控探针

只授权下载最小的一个选定 WT run：`SRR31989028`，paired FASTQ 声明总大小约 4.89 GB。其余两个重复暂不下载。

探针必须：

- 核对两个 FASTQ 的实际大小、ENA MD5 和本地 SHA-256；
- 固定比对工具、版本、索引、完整参数、线程策略、输入顺序和随机性；
- 直接比对到 Strain-B `GCA_001746955.1`，不得写回旧 assembly 坐标；
- 报告 paired/unique/multi/unmapped、覆盖、剪接、四染色体和固定窗口结果；
- 将链方向保持 `unavailable`，不得从正反向 alignment 数量推断文库方向；
- 执行外源/非参考和异常覆盖筛查，并列出不能排除的工程影响区域；
- 双运行主产物逐文件一致，并由不导入项目实现的独立 verifier 重算关键统计；
- 递归保护 Slice 0、0A、0B 和两个已完成来源调查的权威证据。

## 探针通过条件

- 原始文件身份全部通过；
- 四条核染色体均有受控、非零且可解释的覆盖；
- unique、multi、unmapped 和 splice 分类可独立重算，不把未知类别包装为成功；
- 没有全局性外源污染或广泛异常覆盖；局部疑似工程影响被显式隔离；
- 结果只授予其实际具备的能力，不把非链特异数据升级为方向证据；
- 未生成正式边界轨道、安全港候选或阈值。

通过后仍不得自动下载另外两个重复。规划侧需根据探针结果决定是否完成三重复验证。失败时停止并保持 `blocked`。

## 放弃的方案

### 继续机械调查第三个公共来源

前两次调查已表明多个门禁具有系统性而非来源特异性。继续使用同一二元规则只会重复得到 `blocked`，不会增加边界证据。

### 立即下载全部三个重复

总量约 14.85 GB。在工程背景和坐标质量尚未经过受控检查前，先下载全部数据不符合最小可验证切片原则。

### 把未声明链方向的数据完全视为无效

它仍可支持覆盖与剪接，但不能支持方向。整体淘汰会丢失真实能力，整体接受则会过度声明。

## 后果

- `scientific_acceptance_status` 继续保持 `blocked`；
- 项目从“寻找完美单一来源”转向可审计的能力组合；
- 未来边界轨道必须逐能力、逐区域保留来源和限制，不能输出一个掩盖冲突的全局置信标签。
