# ADR-0002：Streamlit 只读展示层

- 状态：Accepted
- 日期：2026-07-14
- 后续修订：[ADR-0010](ADR-0010-streamlit-import-and-trigger-workflow.md) 采纳本 ADR §后果 中预留的"后续若允许网页触发任务，需要新 ADR 替代相关限制"条款，允许网页导入数据并触发核心引擎运行；本 ADR 关于"页面不得复制科学规则、不得直接编辑结果"的核心约束继续有效。

## 背景

研发人员需要通过网页浏览完整基因组分析结果、筛选候选、查看排除理由和核查数据来源。候选筛选本身包含坐标转换、区间集合运算、规则执行和跨参考证据，若在 Streamlit 页面中重复实现，会造成 CLI、文件输出和网页之间的科学结果漂移。

完整基因组分析还可能涉及较大的参考文件和较长运行时间，不适合由页面刷新或交互直接驱动。

## 决策

1. Streamlit 是第一阶段 MVP 的本地网页展示层。
2. 分析核心先生成不可变、版本化且经过完整性校验的运行产物。
3. Run Catalog 只收录 manifest、schema、校验值和跨格式一致性均通过的完成运行。
4. Streamlit 通过展示服务读取 Run Catalog 和规范化结果模型。
5. 页面不得直接导入或调用候选枚举、区间相减、规则执行和共线性计算模块。
6. 首版网页只读，不触发长时间分析、不编辑规则、不修改候选或证据等级。
7. 页面入口、导航注册、页面模块、`st.session_state` 和缓存键属于一个完整变更面，新增、删除或重命名页面时必须一起复审。
8. 页面状态和缓存必须绑定运行 ID 与 manifest 版本；切换运行时清理旧候选选择和派生状态。

## 页面目标

MVP 包含以下用户能力：

- 查看运行概览和 Strain-T 代理适用性声明；
- 浏览、筛选和下载候选窗口；
- 查看单个候选的邻近基因、规则命中和证据来源；
- 查看排除区域及明确理由；
- 查看完整基因组基线统计；
- 查看输入文件、校验值、assembly、annotation、坐标约定和规则快照；
- 识别不完整、损坏或不支持的运行，而不是展示部分结果。

## 放弃的方案

### 页面直接运行完整分析

容易受 Streamlit 重跑语义、页面刷新和会话中断影响，也会将长任务生命周期与 UI 进程耦合。第一阶段不采用。

### 页面与 CLI 分别实现科学规则

会产生不可接受的结果漂移和重复维护。禁止采用。

### 只生成静态报告

实现简单，但无法满足研发人员对筛选、候选详情和排除原因交互浏览的需求。

## 后果

- 分析产物需要稳定 schema 和完成状态。
- 需要独立的 Run Catalog、结果读取器和展示模型。
- UI 可以独立迭代，但不能改变科学结果。
- 页面筛选只改变视图，不改变候选资格。
- 后续若允许网页触发任务，需要新的任务队列、进度、取消和失败恢复决策，并用新 ADR 替代相关限制。

## 验收

- 应用存在唯一入口和唯一权威导航注册。
- 所有页面只依赖展示服务，不直接依赖科学计算引擎。
- `st.session_state` 键集中声明并有默认值。
- 缓存键包含运行 ID 和 manifest 版本或校验值。
- 切换运行不会残留旧候选、筛选或详情状态。
- 页面显示的候选数量、坐标、风险和证据等级与 TSV/JSON 一致。
- 缺失文件、schema 不支持和校验失败会阻止正常展示。
- 本地 Streamlit 健康检查通过，并完成真实浏览器验收。
