# ADR-0005：在锁定主坐标上建立实验数据支持的边界轨道

- 状态：Accepted
- 日期：2026-07-15
- 依据：[ADR-0004](ADR-0004-annotation-qualification-route.md) 与 [Slice 0A 资格报告](../../local_runs/slice0a/qualification_v5_run1/annotation_qualification_report.md)
- 后续修订：[ADR-0009](ADR-0009-deprioritize-proxy-data-compensation.md) 因 Strain-T 原生测序获得授权而暂停本 ADR 的证据整合边界轨道目标与 Slice 0B 后续工作（继续评估新的公开转录组/蛋白组来源）；已完成的 Slice 0B 五轮资格评估（"evidence is insufficient to authorize a full-genome boundary track"）作为历史记录保留，结论继续有效。

## 背景

Slice 0A 已完成来源资格评估。当前 Strain-B INSDC、旧 Strain-B RefSeq 与 Strain-C GenBank 注释均存在系统性边界不确定性；Ensembl 和 KEGG 与旧 ASM2700v1 注释同源，不是独立边界证据；PichiaGenome 2016 整理注释具有 RNA-seq、蛋白质组和人工修订价值，但原始注释文件、精确坐标空间与可复用许可尚未恢复。

旧 Strain-B 到 `GCA_001746955.1` 的四染色体探针证明部分边界可以显式映射，但 36 条探针不足以批准全基因组来源替换，且目标基因邻接仍为 `unavailable`。因此当前没有可直接采用的权威边界来源。

## 决策

1. 保持 Strain-B `GCA_001746955.1` 为唯一主坐标空间，不切换 assembly，也不把它称为 Strain-T 精确坐标。
2. 不直接采用当前任何单一注释来源；不对多个不合格或同源来源进行简单投票式“共识”。
3. 选择建立版本化的“实验数据支持边界轨道”：直接在主坐标上整合合格的转录证据、蛋白同源/蛋白质组证据和可追溯的既有注释先验，并逐基因保留证据来源、冲突与不可用状态。
4. 当前 Strain-B、旧 Strain-B、Strain-C、Ensembl 与 KEGG 注释只能作为待核查先验或功能元数据，不能单独赋予高置信边界。若恢复 PichiaGenome 文件，必须先重新执行来源身份、许可、坐标与映射资格评估。
5. 在完整轨道实现前先执行 Slice 0B“实验数据资格与重注释可行性门禁”。Slice 0B 只决定证据是否足以支撑后续全基因组边界轨道，不生成安全港候选或冻结阈值。
6. Slice 0B 未通过前，`scientific_acceptance_status` 保持 `blocked`；Slice 1 不得开始。

## 证据优先级

从高到低：

1. 可追溯、许可允许且能映射到主坐标的链特异 RNA-seq、长读长转录组或人工验证转录证据；
2. 可追溯的蛋白质组、保守蛋白同源与结构域证据；
3. 已通过身份核验的人工整理注释；
4. 当前与旧版 INSDC/RefSeq 注释、Ensembl 和 KEGG，仅作为先验或元数据。

证据等级不等于自动接受。边界冲突、重复映射、低复杂度、多拷贝与组装端点附近记录必须显式保留，不能用单一结果覆盖。

## Slice 0B 目标与产物

Slice 0B 应产出：

- 版本化实验数据清单：来源、菌株、条件、文库类型、链特异性、读长、重复、登录号、许可、文件大小和 SHA-256；
- 来源独立性与适用性矩阵，区分 Strain-B、Strain-C、Strain-T 和其他近缘菌株；
- 在四条核染色体上的小规模可重复映射/证据探针；
- 污染、唯一映射、剪接位点、链方向、覆盖偏倚和组装端点影响报告；
- “可进入全基因组边界轨道”或“证据不足、继续 blocked”的正式建议与验收清单。

## Slice 0B 验收门槛

- 所有实际使用的数据文件均通过版本、大小和 SHA-256 校验，并记录许可与持久来源；
- 明确每个数据集的菌株和实验条件；未知项不得推断为 Strain-B 或 Strain-T；
- 至少一个独立转录证据来源通过质量与坐标兼容性门禁；若没有，则停止并保持 `blocked`；
- 探针覆盖四条核染色体，并报告成功、冲突、多重映射、不可映射和端点受限类别；
- 相同输入重复运行的主产物逐文件一致，且有未导入项目实现的独立汇总核验；
- 不修改 Slice 0、Slice 0A 的既有证据，不生成候选，不冻结阈值，不实现 Streamlit 页面。

## 放弃的方案

### 直接采用旧 Strain-B RefSeq 或 Ensembl

两者边界身份相同，且旧 RefSeq 的 5040 个 gene 全部带系统性边界不确定性；更换展示来源不能修复科学问题。

### 直接采用 Strain-C GenBank

它属于另一菌株和坐标空间，并同样存在系统性 partial 标记，不能定义 Strain-B 主坐标边界。

### 对现有来源做简单多数共识

多个来源并不独立，简单投票会重复计算同一 ASM2700v1 注释，并把共同错误包装成置信度。

### 立即执行无实验门禁的全基因组从头注释

这会在证据质量、许可和适用菌株尚未明确时扩大范围，且难以建立可审计的边界置信度。

## 后果

- 项目增加 Slice 0B，随后才可能授权全基因组边界轨道切片；
- 科学阻塞被转化为明确的数据资格门禁，而不是通过降低标准解除；
- 后续阈值 ADR 必须引用已验收的边界轨道，不能引用当前 28 个高置信区间或未资格化的注释来源。
