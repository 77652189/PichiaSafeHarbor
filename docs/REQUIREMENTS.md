# PichiaSafeHarbor 需求文档

## 1. 产品目标

本项目面向 `Komagataella phaffii` Strain-T，使用公开参考基因组和注释，在完整核基因组范围内筛选对邻近基因、功能元件和关键基因组结构影响尽可能小的候选整合区域。

项目输出的是“推定安全港候选”（putative genomic safe harbor candidates），用于帮助研发人员缩小实验候选范围。计算结果不得替代湿实验对整合正确性、邻近基因影响、表达能力和长期遗传稳定性的验证。

## 2. 用户与使用场景

主要用户是负责毕赤酵母菌株设计和实验构建的研发人员。

典型流程：

1. 系统加载锁定版本的参考基因组、注释和风险数据。
2. 系统在完整核基因组上枚举原始基因间区。
3. 系统从原始基因间区中扣除受保护功能元件和风险区域，得到可用候选窗口。
4. 系统按 `exclude / flag / annotate` 规则给出处理结果和理由。
5. 系统使用辅助参考检查候选区域的共线性及邻近基因关系。
6. 系统输出候选窗口、被排除区域、分项风险、证据等级和数据缺口。
7. 研发人员通过 Streamlit 网页查看完整运行概览、筛选候选、检查排除理由和数据来源。
8. 研发人员导出选定记录，并在外部成熟工具中完成 sgRNA、同源臂、引物和构建设计。
9. 研发人员通过湿实验验证候选位点。

## 3. 菌株与参考数据

### 3.1 目标菌株

- 目标菌株：Strain-T。
- 第一阶段没有 Strain-T 原生染色体级参考组装，因此结果属于近缘菌株代理推断，不属于 Strain-T 精确坐标结论。
- 2026-07-16 起，项目负责人已授权安排 Strain-T 本身的真实测序，作为长期项目推进；具体测序类型（转录组、基因组重测序或从头组装）与交付时间表尚未确定。在该数据到位并经独立 ADR 评估其对主坐标、注释边界和候选阈值的实际影响之前，不得提前假定近缘菌株代理限制已解除。详见 [ADR-0008](adr/ADR-0008-target-strain-native-sequencing-redirect.md)。
- 同日，[ADR-0009](adr/ADR-0009-deprioritize-proxy-data-compensation.md) 进一步把 Strain-B 明确重新定位为**开发/测试占位数据**：本节其余条款与 3.2-3.4 的 Strain-B/Strain-C 双参考策略在真实数据到位前继续适用，但项目不再投入精力去改善这份代理数据本身的注释或边界可信度（不再追加公开注释来源调查或证据整合边界轨道工作），只维持与数据来源无关的框架代码（参考校验、注释归一化、候选窗口引擎等）持续可用。

### 3.2 主参考

- 菌株：Strain-B。
- assembly：`GCA_001746955.1`（ASM174695v1）。
- 用途：候选枚举、坐标输出和主要风险分析的坐标骨架。
- 已知限制：存在5条未定位 scaffold 和1条未定位质粒；染色体1、3、4在 FASTA/GenBank 中标记为 `partial sequence`，组装元数据声明存在缺失的 3′ 端粒区域。每条染色体的端点完整性必须独立记录，未知端点不得产生保守推荐候选。
- 注释限制：当前 GenBank GFF3 的5288个 gene中有5032个带 `partial=true`，其中5031个同时带 `start_range` 和 `end_range`。这些属性表示注释边界存在系统性不确定性，不能在导入时丢弃。

### 3.3 辅助参考

- 菌株：Strain-C。
- assembly：`GCA_000223565.1`（PicPas_Mar2011）。
- 用途：检查候选区域的共线性、邻近基因关系和结构一致性。

### 3.4 适用性声明

所有输出必须记录：

- `target_strain = Strain-T`
- `primary_reference_strain = Strain-B`
- `primary_assembly = GCA_001746955.1`
- `secondary_reference_strain = Strain-C`
- `secondary_assembly = GCA_000223565.1`
- `strain_applicability = close-strain proxy`
- `exact_target_strain_coordinates = false`

## 4. 第一阶段范围

### 4.1 输入

- 主参考 Strain-B 的 FASTA 和配套 GFF3/GTF。
- 辅助参考 Strain-C 的 FASTA 和配套注释。
- 明确的数据来源、下载日期、校验值和版本标识。
- FASTA/GenBank 中逐序列的 `complete / partial / unknown` 完整性声明。
- GFF3/GTF 中的 `partial`、`start_range`、`end_range` 及等价边界限定属性。
- 可选的重复序列注释。
- 可选的端粒、着丝粒、复制起点和其他结构特征注释。
- 可选的必需基因列表。
- 可选的 Strain-T 相对 Strain-B 的可信变异或结构差异数据。
- 版本化规则配置，包括缓冲距离和候选窗口约束。

### 4.2 分析范围

- 分析主参考的全部已定位核染色体。
- 线粒体序列需要识别和记录，但默认排除出候选集合。
- 未定位 scaffold 和组装不确定区域需要进入审计结果，但默认不进入保守推荐集合。
- 每条已定位核染色体的5′和3′端点完整性必须分别记录；`partial` 或 `unknown` 端点不得按真实端粒距离解释。
- 缺失端粒或未知端点对应的区域必须显式标记，不得视为低风险区域。
- 原始基因间区可以按提交坐标进行描述性统计，但其左右边界必须继承邻近功能实体的边界置信状态。

### 4.3 候选区域模型

系统必须区分两层区间：

1. `intergenic_region`：相邻受保护功能注释之间的原始基因间区。
2. `candidate_window`：从原始基因间区中扣除基因保护缓冲、功能元件、结构风险和其他排除区后得到的连续可用窗口。

如果排除区将一个原始基因间区切分为多个连续窗口，每个窗口必须成为独立候选。第一阶段输出候选区间，不选择具体插入碱基位置。

当任一邻近功能实体带 `partial=true`、开放范围 `start_range/end_range` 或等价不确定边界时：

- 原始区间仍可进入描述性基线统计；
- 必须记录 `left_boundary_confidence` 和 `right_boundary_confidence`；
- 在边界得到独立注释支持或专门 ADR 处理前，不得进入保守候选集合；
- 不得用该区间直接冻结默认基因缓冲距离。

### 4.4 邻近基因方向

- `convergent`：优先候选。
- `tandem`：保留并标记邻近调控风险。
- `divergent`：高风险，默认不进入保守推荐集合。
- `unknown`：证据不完整，不得解释为低风险。

### 4.5 规则动作

规则动作必须分为：

- `exclude`：从候选集合中移除，但保留记录和理由。
- `flag`：保留候选，同时附加风险和证据不足警告。
- `annotate`：记录事实，不直接改变候选资格。

首版硬排除至少包括：

- 与蛋白编码基因、非编码 RNA、tRNA、rRNA、伪基因或其他受保护功能注释重叠；
- 与着丝粒、端粒、重复序列、移动元件或组装缺口重叠；
- 扣除保护区后没有满足最小要求的连续窗口；
- 位于线粒体、未定位 scaffold 或已知坐标不可靠区域；
- FASTA、注释或序列名称不一致，无法证明坐标契约成立。

首版风险标记至少包括：

- 靠近必需基因；
- 邻近基因方向为 `tandem`、`divergent` 或 `unknown`；
- 靠近端粒、着丝粒、重复序列或复制起点；
- 区间异常长，可能包含未注释功能元件；
- 处于 Strain-T 与 Strain-B 可能存在差异的区域；
- 关键风险注释缺失；
- 在辅助参考中无法确认一对一共线关系。

### 4.6 阈值确定

第一阶段不得直接套用哺乳动物或其他酵母的距离阈值。

实现顺序必须先完成完整基因组基线统计，包括基因间区长度、邻近基因方向、受保护注释覆盖和候选窗口数量分布；再根据数据分布提出默认缓冲距离与最小窗口长度。阈值冻结属于独立架构决策，需另建 ADR。

阈值 ADR 的前置条件还包括：注释边界适用性门禁通过。必须分别报告全部提交坐标和高置信边界子集；如果高置信子集不足以支持阈值推断，必须先更换、补充或协调注释来源。

当前 Strain-B GenBank 注释的高置信双边界区间仅28/4220，不足以代表完整基因组，也不得作为默认阈值依据。项目必须保留 `GCA_001746955.1` 主坐标并先完成独立注释来源资格评估；不得仅使用这28个区间继续正式候选分析。

### 4.7 输出

至少生成：

- `integration_site_candidates.tsv`
- `integration_site_candidates.json`
- `excluded_regions.tsv`
- 候选筛选报告（Markdown 或 HTML）
- 运行元数据与规则快照
- 独立验收清单
- Streamlit 网页展示

每个候选至少包含：

- `candidate_id`
- 目标菌株、主参考和辅助参考版本
- 染色体、起止坐标和坐标约定
- 原始基因间区与候选窗口坐标
- 候选窗口长度和序列
- 左右邻近基因、方向和距离
- 功能元件、必需基因、重复序列、端粒和着丝粒风险
- 辅助参考共线性状态
- Strain-T 适用性及变异风险状态
- 邻近干扰、结构稳定性和证据完整度等分项评价
- `validated / literature-supported / predicted / unavailable` 证据等级
- `exclude / flag / annotate` 规则结果、理由和来源
- 数据、规则和参数版本

运行状态必须拆分为：

- `execution_status`：计算过程是否完成；
- `verification_status`：自动化和独立验证是否通过；
- `scientific_acceptance_status`：当前切片是否满足科学验收，可取 `pending / accepted / blocked / rejected`。

不得用单一 `status=complete` 同时表达计算完成和科学验收完成。

### 4.8 Streamlit 网页展示与数据导入触发

MVP 必须提供本地 Streamlit 网页，用于查看已经完成且通过完整性校验的分析运行，并允许导入新参考数据、触发核心引擎运行（[ADR-0010](adr/ADR-0010-streamlit-import-and-trigger-workflow.md)）。网页不是独立的科学计算引擎：触发运行时只能调用核心库已有的顶层函数（如参考校验、`run_baseline`、`run_slice1`），不得在页面代码中重新实现候选枚举、区间相减、风险判定或证据分级。

网页至少提供：

- 数据导入与运行触发：导入/指向新的参考 bundle，触发 baseline、candidate-windows 等运行，并展示执行状态（运行中/完成/失败）；
- 运行概览：目标菌株、主/辅参考、数据版本、规则版本、候选数量、排除数量和数据缺口；
- 候选浏览：按染色体、邻近基因方向、风险标记、证据等级和共线性状态筛选；
- 候选详情：坐标、序列、左右邻近基因、规则命中、排除区切分来源和证据来源；
- 排除记录：查看被排除区域、对应规则、参数和理由；
- 基因组统计：基因间区长度和邻近方向分布等基线图表；
- 数据与方法：输入文件、校验值、assembly、annotation、坐标约定、规则快照和已知限制；
- 下载：导出当前筛选结果或原始版本化 TSV/JSON，不改变科学结果。

网页必须显著展示：

- `target_strain = Strain-T`
- `primary_reference_strain = Strain-B`
- `exact_target_strain_coordinates = false`
- 当前运行是否完整、是否通过 schema 和跨格式一致性校验
- 当前运行的计算、验证和科学验收三类状态

网页可以触发核心引擎生成新的版本化运行，但不得编辑规则或直接修改已有运行的结果文件；触发的新运行遵循与命令行完全相同的产物格式、`run_id` 生成方式和三态状态模型。网页刷新和筛选不得改变已有候选集合。

## 5. 第一阶段不包含

- sgRNA、CRISPR/Cas、同源臂和引物设计。
- 载体、启动子、终止子或完整表达盒设计。
- 湿实验构建和实验数据采集。
- KO/OE 分泌机制、代谢流或生长权衡预测。
- 构建或宣称拥有 Strain-T 原生完整参考基因组。
- 将 Strain-B 坐标无条件等同于 Strain-T 坐标。
- 将文献中的高效整合位点直接认定为安全港。
- 用单一黑盒总分掩盖安全性、表达潜力和证据完整度差异。
- 首版支持多个目标物种。
- 在 Streamlit 页面中复制或绕过核心科学规则。
- 通过网页直接编辑已完成运行的候选、排除理由或证据等级。

## 6. 产品与架构边界

- 本项目是独立项目，不依赖 `pcSecYeastSpecies` 的 LP、GPR 或 KO/OE 求解代码。
- 与其他项目通过版本化 TSV/JSON 契约交互，不跨仓库导入源码。
- 运行产物与源代码、固定测试数据分开管理。
- 科学判断必须可追溯，规则、参数、来源和排除理由必须进入结果元数据。
- 安全风险、表达环境、结构稳定性和证据完整度必须保持为独立维度。

## 7. MVP 验收标准

使用锁定版本的 Strain-B 和 Strain-C 完整参考数据，系统应能够：

1. 可重复地处理全部已定位核染色体，并明确排除线粒体和未定位 scaffold。
2. 正确枚举原始基因间区及左右邻近基因、方向和距离。
3. 根据版本化规则生成一个或多个候选窗口，并记录所有区间切分来源。
4. 输出候选与被排除区域，并为每条记录给出明确动作和理由。
5. 在 TSV、JSON 和报告之间保持候选数量、坐标、证据等级和风险标记一致。
6. 记录数据来源、校验值、assembly、annotation、规则和参数版本。
7. 对坐标边界、正负链、重叠注释、染色体端点完整性、未定位 scaffold、`partial/start_range/end_range` 和缺失注释进行自动化验证。
8. 对同一输入重复运行得到完全一致的候选集合。
9. 对候选执行 Strain-C 共线性核查，并区分 `confirmed / conflicting / unavailable`。
10. 对至少一组文献报道整合区域进行回顾性检查，明确区分“可整合证据”和“安全港验证”。
11. 报告中显著展示 `exact_target_strain_coordinates = false`，不得让用户误解为 Strain-T 原生坐标。
12. Streamlit 能加载一个完整运行，并展示与 TSV/JSON 一致的候选数量、坐标、风险和证据等级。
13. 页面导航、候选筛选、候选详情、排除记录、统计图表、数据来源和下载流程可用。
14. 页面缺少运行文件、schema 不匹配或运行不完整时明确阻止展示，不产生部分或推测性结果。
15. 页面刷新、筛选和下载不修改源结果，且 `st.session_state` 与缓存键不会在切换运行后泄漏旧运行状态。
16. 正式 acceptance manifest 绑定实际输入哈希、运行 ID、产物哈希、独立验证结果和科学验收结论。
17. 只有 `scientific_acceptance_status = accepted` 的运行才能作为后续阈值 ADR 或正式候选分析的输入。
18. 网页触发的运行与命令行触发的运行遵循同一套 RunManifest/AcceptanceManifest 契约和三态状态模型，不产生弱化版产物格式；网页不提供编辑已有候选、排除理由或证据等级的功能。

## 8. 后续阶段候选

- 引入可靠的 Strain-T 变异、结构变异或实验室自有测序数据。
- 构建经验证的 Strain-T 共识基因组或原生组装。
- 引入 RNA-seq、位置效应或染色质证据，独立评价表达环境。
- 建立文献支持位点库和实验反馈闭环。
- 在需求稳定后评估扩展到其他菌株或酵母物种。
- 向统一研发工作台提供稳定的文件导入或 API 契约。
